# BTC Price-Only 长样本结论报告

这份报告只回答一个问题：

**现在这条 `BTC price-only basis` 策略，到底有没有继续往实盘推进的价值？**

先说结论，再说理由。

## 一句话结论

这条策略现在已经不是“完全靠小样本碰运气”的状态了。

从研究回测角度看，它**大概率确实抓到了某种可重复的定价偏差**，值得继续做。

但如果问题是“现在能不能直接上自动实盘”，我的答案还是：

**还不能。**

原因不是它完全没 edge，而是：

- 样本虽然比之前大很多，但仍然明显集中在几个事件簇
- 全样本收益非常夸张，必须先当成“需要怀疑的好结果”
- 当前回测执行和真实成交之间，还隔着一层研究近似

所以更准确的说法是：

**这条策略已经从“想法”进化成了“值得认真继续做的研究策略”，但还没进化成“可以放心自动上真钱”的实盘策略。**

## 这次到底检查了什么

这次不是拿之前那份 3 个事件的小样本继续说故事，而是把样本重新扩了一轮，并把两个会污染结论的问题先修掉了。

先修的两个问题：

1. `DuckDB 1.5.0` 在这里对 `PRIMARY KEY` 表做大批量 `COPY` 会直接崩进程。现在改成了“先安全导入，再显式校验唯一性”。
2. 样本发现范围比策略支持范围更宽，混进了 `Will Bitcoin hit $100k again in 2024?` 这种题面。现在构库阶段就筛掉，只保留策略真正支持的 `above / below / between`。

修完以后，这次最终用来评估的样本是：

- 时间范围：`2025-01-10 20:45 UTC` 到 `2026-03-18 19:20 UTC`
- 独立事件数：`46`
- 市场数：`200`
- token 数：`400`
- token-分钟观测数：`3,304,119`

注意，这里说的不是“搜到了多少 BTC 相关题”，而是**真正留在最终可回放样本里的、策略支持的价格型题面**。

## 全样本结果

全样本默认保守参数回测结果如下：

- 初始资金：`100,000 USD`
- 期末权益：`526,049.67 USD`
- 组合净收益：`+426,049.67 USD`
- 总收益率：`+426.05%`
- 年化夏普：`4.41`
- 最大回撤：`25,899.92 USD`
- 往返次数：`5,987`
- 胜率：`63.99%`

如果只看这张表，结论会非常诱人，甚至有点“好得不正常”。

所以真正重要的问题不是“它赚了多少”，而是：

**这个收益是不是太依赖少数阶段、少数事件，或者某种研究口径带来的乐观偏差？**

## 哪些地方说明它不是纯碰运气

先讲正面的部分。

这次结果里，确实有几条信息支持“它可能有真实 edge”：

- `200` 个市场里，有 `148` 个市场赚钱，占 `74%`
- `46` 个独立事件里，有 `40` 个事件赚钱，只有 `6` 个事件亏钱
- 单一最大事件只贡献了总收益的 `13.44%`
- 前 `5` 个事件一共贡献 `39.91%`
- 前 `10` 个事件一共贡献 `59.52%`

这说明一件事：

**利润不是只靠一个市场、一个事件、一次运气撑起来的。**

如果它真的是单点碰运气，通常会看到：

- 头部 1 到 2 个事件吃掉绝大部分利润
- 大量市场本身并不赚钱
- 事件层面正负分布很差

而这次不是这个样子。

换句话说，站在“有没有继续研究价值”的角度，我认为答案已经是：

**有。**

## 哪些地方说明还不能轻易相信它已经稳了

这里是更重要的部分。

### 1. 收益明显集中在几个时间簇

按事件结算月份粗看，收益分布是这样的：

- `2025-08`：约 `+155,482 USD`
- `2025-09`：约 `+145,062 USD`

这两个月合起来，大约占总收益的：

- `70.54%`

也就是说，虽然利润不是只靠一个事件撑起来的，但它**非常明显地集中在 2025 年 8 月和 9 月那一波 BTC 价格型市场簇里**。

这代表两个现实：

1. 策略可能确实适合某一类市场环境
2. 但它还没有证明自己跨环境一样稳定

如果把 `2025-08` 和 `2025-09` 整段拿掉，全样本仍然是正收益，但只剩大约：

- `+125,506 USD`

这说明它不是完全离开这两个月就死掉，但也说明：

**目前看到的大幅赚钱，很大一部分来自少数高质量阶段。**

### 2. 样本外结果是正的，但不能直接当连续实盘权益

我跑了一版 `60 天训练 / 14 天测试 / 14 天步长` 的 walk-forward。

结果表面上非常漂亮：

- 测试切片数：`6`
- 盈利测试切片占比：`100%`
- 每段测试都是正收益

但这里有一个必须讲清楚的限制：

**这些测试窗不是连续自然日历月份，而是沿着“有可交易样本的事件簇”往前滚。**

比如某段测试在 `2025-05`，下一段可能直接跳到 `2025-07`，再跳到 `2025-08`。

这意味着：

- 它可以说明“在多个事件簇里，这套固定参数都赚到过钱”
- 但它**不能**直接等价成“如果你从头到尾连续实盘跑，权益就会长成那样”

所以这里正确的理解应该是：

**样本外方向是偏正面的，但强度还不能按报表字面意思去信。**

### 3. 回测收益仍然不是实盘可成交收益

这个边界没有变，必须反复强调：

- Poly 腿虽然按真实 premium 记账
- 但 CEX 腿仍然是研究版数值 delta 近似
- Polymarket 真下单执行还没有接完
- 真实盘口深度、成交延迟、排队位置、部分成交这些问题还没有被完整模拟

所以这份回测更适合回答：

**“有没有研究价值”**

而不是直接回答：

**“上真钱以后能不能赚这么多”**

## 现在该怎么判断它

如果一定要给一个很直白的评级，我会这样打：

- 是否有继续做的价值：`有，而且不低`
- 是否已经证明自己稳定：`还没有`
- 是否可以继续 paper / shadow run：`可以`
- 是否适合直接自动实盘：`不建议`

再讲得更大白话一点：

这套东西现在已经过了“拍脑袋阶段”。

它不是那种一测就露馅的假 alpha。

但它也还没有过“实盘前最后那几道坎”：

- 长样本是否跨环境稳定
- 回测执行链和未来实盘链是否完全同源
- 真实成交以后，收益会不会被滑点和延迟吃掉

## 我建议的下一步

如果目标还是“朝实盘推进”，我建议按这个顺序继续：

1. **把历史回放并到 Rust 主引擎**
   现在 `sim/paper` 是 Rust，`backtest` 还是 Python 主循环。
   先统一事实源，避免“回测一套、实盘一套”。

2. **做连续日历切片的样本外验证**
   不是只沿着事件簇切，而是按连续月份去看。
   这样更接近真实上线以后会经历的市场时间。

3. **补执行层压力测试**
   把成交延迟、盘口深度、部分成交、额外滑点都压进来。

4. **继续 paper run，但别急着真钱自动化**
   现阶段最合理的定位是：`认真 paper，不急 live`。

## 最后一句话

如果你问我：

我的回答是：

**值得。**

如果你问我：

**“现在就能不能放心自动实盘？”**

我的回答是：
